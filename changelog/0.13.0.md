## [0.13.0] — 2026-05-18 · Tabular Editor cleanup script alongside the AI prompt

The v0.12.0 AI cleanup prompt is great if you already use Claude Code. Many BI devs live in Tabular Editor instead — v0.13.0 generates a paste-into-TE C# script as a second output format from the same audit data, no AI loop required.

### Added
- **`buildTabularEditorScript(data, category)` in `src/ai-prompts.ts`** — emits a `.csx` script that paste-runs in both Tabular Editor 2 (free) and Tabular Editor 3 (paid). Uses anonymous-type target entries (C# 3.0-compatible) so it works across every TE version. Same Stage 1 / Stage 2 ordering as the AI prompt; same EXTERNALMEASURE + auto-date filtering, with an *additional* runtime EXTERNALMEASURE guard in the script body as defence-in-depth.
- **Modal format toggle** — the cleanup-export modal now has a segmented control at the top: **AI prompt (.md)** \| **Tabular Editor script (.csx)**. Toggle re-renders the body, swaps the hint text, and updates the download button label / filename extension.
- **CLI `--format` flag** — `powerbi-lineage prompt --format te-script ...` emits the TE script to stdout / file. Default is `ai-prompt` (backwards compatible with v0.12.0). Unknown values fail loudly.

### Script anatomy
The TE script:
1. Hardcodes the target list as anonymous-type entries with `{ Table, Measure, Stage }`.
2. Iterates with safety guards: missing table → skip, missing measure → skip (idempotent — re-runs after partial saves do the right thing), EXTERNALMEASURE in expression → skip.
3. Calls `m.Delete()` only after all checks pass.
4. Prints a tally and reminds the user to Ctrl+S (TE doesn't auto-save).

### Changed
- **Modal title is now generic** ("Cleanup export") since it shows both formats. Subtitle adapts per-format-per-category ("AI cleanup prompt — directly unused measures" / "Tabular Editor cleanup script — unused + dead-chain measures" / etc.).

### Tests
- **311 tests pass** (was 299): 9 new for the TE script generator (header, target entries, EXTERNALMEASURE static + runtime, Stage ordering, auto-date exclusion, no-op empty case, C# string escaping, trailing newline, Ctrl+S reminder), 2 new for the CLI `--format` flag. No new runtime dependencies.

### Notes
- **TE2 vs TE3**: the script targets both. Anonymous types (C# 3.0), string interpolation (C# 6), and `Output(...)` / `Model.Tables.FindByName(...)` are all supported in both versions.
- **No-op empty case**: if a category has zero candidates, the script emits a single `Output(...)` line announcing nothing to do — so users running it via automation don't trip on a missing `targets` array.
- The AI prompt path is untouched. Existing CLI invocations without `--format` continue to emit markdown.
