## [0.10.1] — 2026-04-25 · Cross-doc links (F13) + F6 deferred-with-rationale

Closes the last two deferred items from the 0.10.0 design doc. F13 ships as a meaningful navigation upgrade across the Detailed-mode docs; F6 is dropped after re-examination shows the post-cleanup repetition is mostly inside raw-M bodies that can't be aliased without lying about the source.

### Added

- **Cross-doc xref helper (F13).** New `xref(text, doc, anchor)` in `md-generator.ts` emits a markdown link `[text](DocName.md#anchor)` for cross-doc references, or `[text](#anchor)` when `doc === null` for same-doc anchors. Anchor passes through `adoSlug()` so it lines up with what the destination doc actually emits.

  Applied at:

  | Source location | Links to | Why |
  |---|---|---|
  | `Measures.md` per-measure `**Table:**` field | `DataDictionary.md#table-name` | Jump from a measure to its home table's schema |
  | `Measures.md` Lite summary table cells | `DataDictionary.md#table-name` | Same, in the flat A–Z view |
  | `Functions.md` "Used by" caller chips | `Measures.md#measure-name` | Jump from a UDF to each calling measure |
  | `Pages.md` visual binding cells | `Measures.md` / `DataDictionary.md` | Per-binding deep-link from a page to the entity |
  | `Sources.md` physical-source index "Model tables" | `DataDictionary.md#table-name` | Jump from "what feeds this source?" to the table schema |
  | `DataDictionary.md` FK / Ref chips | `#table-name` (same-doc) | Click an FK target to jump to its section |

  **Renderer compatibility:**
  - **GitHub** — relative file refs + anchors resolve cleanly when the docs sit in the same folder. ✅
  - **ADO Wiki** — file extensions strip on paste; users with `.md` in URLs need find/replace. Compat table in README will be updated.
  - **Dashboard MD viewer** — new click-intercept in `main.ts` detects `*.md#anchor` hrefs in rendered docs, tab-switches to the named doc, then scrolls to the anchor on the next tick. Same-doc `#anchor` links keep their existing scroll-only behaviour.

- **Anchor safety guards.** The DataDict FK/Ref xref only emits a link when the target table actually has a section in DataDict (i.e., is a user table) — auto-date infrastructure tables collapse to plain text so we don't generate broken anchors. Caught by an existing test.

- **Lite mode preserves cross-link discipline.** Lite Sources.md does NOT emit DataDictionary links (DataDictionary doc is skipped in Lite — links would 404 on paste). Detailed Sources.md links freely.

### Changed

- **Default `mdMode` reset on report swap.** The `__loadBrowserData` hook now resets `mdMode = "detailed"` alongside the other UI state (search terms, sort directions, etc.) so a stale Lite/Detailed selection from a previous report doesn't carry into the new one. This was a missed reset in 0.10.0.

### Tests

- **+6 F13 tests** asserting:
  - Detailed Sources.md links Model tables to DataDictionary
  - Lite Sources.md does NOT link (DataDictionary skipped in Lite)
  - Detailed Measures.md links per-measure home table
  - Detailed Pages.md links visual bindings
  - Detailed DataDict FK chips emit same-doc links
  - DataDict FK pointing at auto-date table falls back to plain text (no broken anchor)

### Deferred (not landing)

- **F6 — Source-alias pattern.** Re-examined post-PR-#83 + #85 cleanups: 16 URL mentions per cluster on the H&S fixture, but **14 of those are inside raw-M expression bodies** in the Raw M Expressions section. Substituting `[AS-1]` for the URL inside a raw M body would lie about what the M code actually says — the body is meant to be the source of truth. Only 2 reducible sites remain (bucket sub-line + physical-source-index row), which is below the bar to justify introducing the alias-key apparatus.

  **Decision: F6 dropped without a follow-up.** The "60+ URL mentions" diagnosis from the original `/sc:analyze` audit predated PR #83 (M-step shape collapse) and PR #85 (§3.3 removal). After those landed, F6's surface area shrank to two-line wins that don't justify the reader-confusion of an alias key.
