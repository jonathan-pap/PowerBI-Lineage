## [0.8.1] — 2026-04-24 · Post-/sc:analyze hardening (P1–P4 shipped)

Infrastructure + typing pass following the post-0.8.0 analyse-skill review. No user-visible behaviour changes — bundle ships the same pixels. Four PRs land the four prioritised recommendations from the analysis report.

### Added

- **Bundle-size CI guard** (PR #55) — `.github/workflows/ci.yml` now measures `docs/index.html` after `npm run build:browser` and fails if it exceeds 300 KB. Current size 234 KB, so 28% headroom for organic growth; regressions from accidentally-inlined vendor assets or oversized payloads surface on the PR before merge.
- **New module `src/browser/pair-picker.ts`** (PR #56) — 370 lines of pure logic extracted from `entry.ts`'s `showPairPicker()`. Zero DOM imports. Exports `scanPairCandidates`, `validatePair`, `findBestModelForReport`, `findBestReportForModel`, `filterAndRemount`, `installModelOnlyShim`, `installReportOnlyShim`, `computePickerDefaults`, `buildPickerCardHtml`, `classifyLoadMode`, plus types. Orchestrator stays in `entry.ts`.
- **37 new unit tests** (PR #56) — `tests/browser-pair-picker.test.ts` covers every branch in the extracted module without a DOM. Total now **194 tests** (was 157). Runs in ~1.2 s.

### Changed

- **`src/client/main.ts` compiles under strict mode** (PR #57) — `@ts-nocheck` removed. 400 latent type errors worked through in five phases: DOM lib + ambient decls → `: any` on 113 callback params → null-safety casts / non-null assertions on 50 DOM sites → typed `Record<…>` index signatures on module state → narrow casts for `e.target.value` / `setSelectionRange`. No runtime change (every fix is either type-erased or a compile-time cast).
- **`showPairPicker()` shrank 326 → 129 LOC** (PR #56) as a side-effect of P1's extraction. The orchestrator now pastes HTML built by a pure helper, installs DOM accessors, wires listeners that delegate to pure-function decisions.
- **`globals.d.ts` grew ambient declarations** for `escHtml`, `escAttr`, `sc`, `uc` (from `render/escape.ts`) and `mdRender`, `mdEscapeHtml`, `mdInline` (from `render/md.ts`) — these are script-concatenated into the same inline `<script>` as main.js, not imported. Declarations make the concatenation pattern type-safe.
- **README refreshed** — test-count badge + prose updated 157 → 194; project-layout section expands `client/` and `browser/` to list their key files (including the new `pair-picker.ts`).
- **Landing overlay trust chip** updated 157 → 194 tests.

### Infrastructure

- `tsconfig.json` unchanged — the DOM lib pull-in lives in `src/client/globals.d.ts` via `/// <reference lib="dom" />` directives, keeping server-side files (app.ts, data-builder.ts) free of accidental DOM type exposure.
- `DATA` and `MARKDOWN_*` declarations flipped from `declare const` → `declare let` to match the PR #34 runtime change (browser-mode reassigns them via `__loadBrowserData`).
- Added `declare let MARKDOWN_CHANGELOG` (previously missing — slot added for PR #51 but never declared).

### Verification

- `npm run typecheck` → 0 errors (was 400 with `@ts-nocheck` removed)
- `npm test` → **194/194 green** across Node 18 / 20 / 22
- `npm run build:browser` → clean; bundle **234 KB** (under 300 KB CI cap)

### Not included

- P-lite "partial typing" — we took the full P4 path instead, so all of main.ts is strict-clean now.
- User-visible feature work — intentionally deferred. 0.8.1 is an infrastructure release; user-visible additions resume in a later 0.9.x.
