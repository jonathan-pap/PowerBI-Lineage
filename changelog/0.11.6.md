## [0.11.6] — 2026-04-26 · Cross-doc link integrity (H1 + H3 from MD-quality audit)

A targeted fix for two cross-doc-linking issues surfaced by `/sc:analyze markdown docs`. Both shipped together because the test extension that catches one catches the other.

### Fixed
- **`Model.md` §3.3 emitted `[Sources.md](#)`** — link href was the empty fragment `#`, scrolling to top-of-page on every render target instead of jumping to the Sources companion doc. Now correctly emits `[Sources.md](Sources.md)`.

### Changed
- **`Index.md` is no longer a dead-end glossary.** Every entry now links to the companion doc that owns its full reference:
  - **Tables / Calc groups** → `DataDictionary.md#<table>`
  - **Measures** → `Measures.md#<measure>` (existing `<a id>` anchor)
  - **Columns** → `DataDictionary.md#<parent-table>` (no per-column anchor today; lands on the owning table — best available target)
  - **UDFs** → `Functions.md` (doc-level link; per-UDF anchors are a separate follow-up)
  - **Calc items** → `CalcGroups.md#<parent-cg>`
- **`xref()` helper hardened.** When called with an empty `anchor`, the helper used to silently emit `[text](Doc.md#)` — same regression class as H1. It now emits `[text](Doc.md)` (no fragment).

### Tests
- **8 new "no empty-fragment links" tests** — one per generated doc (Model / Measures / Functions / CalcGroups / DataDictionary / Sources / Pages / Index). Catches `[text](#)` and `[text](Doc.md#)` patterns that resolve to top-of-page everywhere.
- **1 new Index cross-linking test** — asserts `Index.md` contains at least one link into `Measures.md` and one into `DataDictionary.md`. Without it the glossary regresses to a flat list.
- Test count: **267 → 276**.

### Notes
- This release closes the High-severity findings from the MD-quality audit. Medium / Low findings (Improvements `items` enrichment, anchor uniqueness, dead Mermaid helpers) are tracked but not in scope here.
